# R1a Three-Sphere Research Ops Case Report

## Working hypothesis

Research organizations benefit from separating domain questions, decision-facing interpretation, and reusable technical infrastructure into distinct but linked spheres.

## Design decisions captured here

- `SCIENCE` holds domain claims and study logic
- `ENTREPRENEURSHIP` holds translation, ecosystem, and public-case framing
- `TECHNOLOGY` holds reusable methods, scoring, interfaces, and registries
- `Hugging Face` acts as the interactive layer, while `GitHub` remains the structural source of truth

## Evidence to preserve

- repository-level README changes
- starter studies added to `E` and `T`
- shared registry data model
- synchronization and visibility improvements
