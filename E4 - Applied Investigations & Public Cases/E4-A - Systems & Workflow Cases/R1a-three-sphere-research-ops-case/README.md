# R1a Three-Sphere Research Ops Case

## Question

How should K R&D Lab structure research, tooling, and public-facing artifacts across GitHub and Hugging Face so the whole ecosystem stays navigable, testable, and reusable?

## Scope

This is a real public case about research operations and systems design across the three spheres.

## Case focus

- separating domain research from reusable tooling
- deciding what belongs in `SCIENCE`, `ENTREPRENEURSHIP`, and `TECHNOLOGY`
- keeping GitHub as the source of truth and Hugging Face as the interactive layer
- making the org legible through starter studies, README maps, and shared registry logic

## What changed in this case

1. `SPHERE-I-SCIENCE` received clearer lane boundaries and conference-aligned expertise mapping.
2. `SPHERE-II-ENTREPRENEURSHIP` gained starter studies for opportunity mapping, audience segmentation, and ecosystem mapping.
3. `SPHERE-III-TECHNOLOGY` gained starter studies for pipelines, scoring systems, and registry/dashboard work.
4. A first cross-sphere registry scaffold was added to support unified browsing.

## Why it matters

This case turns the lab from a set of disconnected repositories into a more coherent research operating system.

## Status

Public systems case scaffold based on active repository restructuring.
