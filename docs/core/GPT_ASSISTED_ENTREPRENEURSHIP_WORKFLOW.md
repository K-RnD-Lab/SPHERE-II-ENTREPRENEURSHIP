# GPT-Assisted Entrepreneurship Workflow

## Short answer

Yes, entrepreneurship files from this repo can be sent directly into GPT chats.

The best format is a compact venture or public-case pack:

- starter study README
- relevant execution index section
- evidence or source notes
- assumptions
- expected output skeleton

## What to send into GPT

For one entrepreneurship route, send:

1. the starter study `README.md`
2. `docs/core/E1_E4_RESEARCH_EXECUTION_INDEX.md`
3. any existing notes, market signals, or ecosystem observations
4. the output you want: report, opportunity map, ecosystem map, public case, or validation guide

If no skeleton exists yet, ask GPT to create one first.

## Recommended chat sequence

### Step 1: frame the route

Ask GPT:

```text
Read this entrepreneurship route. Do not write the final output yet. First identify the decision question, stakeholder, evidence needed, assumptions, and likely repo artifact.
```

### Step 2: define evidence

Ask GPT:

```text
List the evidence needed to make this entrepreneurship claim credible. Separate market evidence, ecosystem evidence, user/customer evidence, operating evidence, and missing evidence. Do not invent facts.
```

### Step 3: draft the artifact

Ask GPT:

```text
Draft a repo-ready Markdown artifact. Include problem or opportunity framing, stakeholder, assumptions, evidence needs, risks, next action, and source list.
```

### Step 4: make it usable

Ask GPT:

```text
Make this useful for a founder, operator, or research lead. Remove generic startup language. Keep only specific decisions, checks, and next actions.
```

## Quality rules

Do not accept a GPT-written entrepreneurship output unless it has:

- clear decision or opportunity question
- target stakeholder
- evidence needs
- assumptions
- risks and limits
- practical next action

Smooth persuasive text is not enough.
