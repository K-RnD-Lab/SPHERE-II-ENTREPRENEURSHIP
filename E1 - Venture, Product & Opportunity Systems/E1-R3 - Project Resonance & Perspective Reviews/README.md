# E1-R3 - Project Resonance & Perspective Reviews

This research line is separate from `E1-R2 - Product & Venture Validation`.

`E1-R2` is for project roasts: critique, stress-testing, scores, recommendations, and action plans.

`E1-R3` is for resonance and perspective reviews: argued personal impressions, associations, belief signals, future potential, and whether a project feels worth watching, supporting, testing, or avoiding.

## Current artifacts

- [R3 Project Resonance Review Methodology](./R3-project-resonance-review-methodology/)
- [R3a ISR Project Resonance Review Draft](./R3a-isr-project-resonance-review-draft/)

## Boundary rule

Use `E1-R3` when the output should read like a rigorous perspective essay rather than a founder-facing improvement memo.

The review can be critical and sharp, but it should not become a recommendation checklist unless it is intentionally moved into `E1-R2`.
