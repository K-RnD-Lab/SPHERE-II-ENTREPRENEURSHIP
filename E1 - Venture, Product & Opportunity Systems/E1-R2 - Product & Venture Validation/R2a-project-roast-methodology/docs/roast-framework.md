# Project Roast Framework

## Method summary

The K R&D Lab project roast combines:

- venture validation
- product positioning
- cold-user journey review
- evidence and credibility audit
- social-impact interpretation
- practical implementation planning

It is designed for early projects that are promising but not yet fully legible to outsiders.

## Diagnostic axes

### 1. Commercial readiness

Checks whether the project has a plausible path to adoption.

Signals:

- clear user or buyer
- clear pain point
- plausible pilot path
- credible funding or revenue logic
- visible reason to act now
- competitor or alternative awareness

### 2. Cold-user readiness

Checks whether an outsider understands the project quickly.

Signals:

- homepage explains the project in plain language
- demo or visual proof appears early
- jargon is controlled
- CTA is clear
- first-time visitor can answer: "What is this, who is it for, and why should I care?"

### 3. Proof readiness

Checks whether the project can support its own claims.

Signals:

- public demo
- pilot data
- case study
- source documentation
- method explanation
- limitation statement
- responsible disclaimers

### 4. Social-impact readiness

Checks whether the project has a responsible public-value story.

Signals:

- affected groups are named
- likely benefits are concrete
- likely harms or exclusions are acknowledged
- governance or accountability is visible
- adoption risks are not hidden behind optimism

### 5. Execution readiness

Checks whether the next version is realistic.

Signals:

- MVP scope is small enough
- roadmap is sequenced
- the first user action is testable
- technical dependencies are visible
- operational risks are named

## Typical recommendation types

- rewrite the hero message
- move demo above philosophy
- separate research, product, and grant pages
- create a pilot one-pager
- add a 3-minute preset demo
- define ICP and non-ICP
- add social-impact assumptions
- add evidence and limitation blocks
- reduce jargon tax
- define the next pilot conversation

## Output format

A roast should produce:

- scorecard
- key diagnosis
- prioritized action plan
- optional landing-page copy
- optional social-impact map
- optional pilot or grant framing

## Tone rule

The tone can be sharp, funny, and memorable.

The substance must remain useful, fair, and evidence-facing.

Do not confuse theatrical critique with real diagnosis.
